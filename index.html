HTML Forms & CSS Display Cheat Sheet

HTML Forms

Form Tags

Element Description Example
<form> Represents section of the document containing control for information submission <form></form>
<fieldset> Groups controls and labels within the form element <form>
<fieldset></fieldset>
</form>
<legend> Gives a caption for the fieldset element <fieldset>
<legend>Select One:</legend>
</fieldset>
<label> Gives a caption for an item on a form that is within the fieldset <fieldset>
<label>Enter Your Name</label></fieldset>
<input> Creates interactive controls like a textbox or selection bubble <fieldset>
<label>Enter Your Name</label> <input />
</fieldset>
<select> Creates a dropdown menu <fieldset>
<label>Choose One:</label>
<select></select>
</fieldset>
<option> Defines content within the select element <select>
<option></option>
</select>
<textarea> Creates a freeform area for text to be entered <fieldset>
<label></label>
<textarea></textarea>
</fieldset>
Things to Note:
  • <label> and <input> always go together and are linked by for and id attributes.
  • <input> is defined by type.
  • <option> has to have a specified value.

Form Attributes

Name Description Example
action Tells browsers the URL or relative path that submissions are sent to <form action=""></form>
method Defines how data is sent <form action="" method="get"></form>
type Tells browser what date to expect (i.e. text, checkbox, radio, email, file, etc.) <input type="text" />
radio Generally used for describing radio group options <input type="radio" />
checked Auto selects an option <input type="radio" checked />
pattern Defines inputs for text, tel, email, url, password, and search types <input type="password" pattern="[a-z0-5]{8,} />
placeholder Can be used with any input to display holder text <input type="text" placeholder="placeholder text here" />
Things to Note:
  • Most common method values are GET )sends back a server response at submission( and POST (sends submission to server and asks for a response back).
  • Types can include submit, text, checkbox, radio, file, password, email, etc.
  • Only one radio input can be selected as long as they're connected by the same name and value.

CSS Display & Postioning

Property Description
position: relative Places elements relative to thier default posisitions
position: absolute Places elements relative to thier closest parent
position: fixed Places an element to a certain point on the page that does not change when scrolling
position: sticky Keeps element placed in its normal flow until the page hits a certain point, then it becomes fixed
display: inline Makes elements next to each other
display: block Makes an element fill the entire width of the screen or a set width
display: inline-block Combines features so that elements appear next to each other but can also have specfied dimensions
z-index Specifies the depth )z-axis( of the element on the screen to the user, as well as if the element uses local stacking.
float Places element to the left or right of it's container
(Commonly used to wrap text around an image)
clear Determines how elements behave when they bump into each other.
Things to Note:
  • Positions are specifed by using the top, bottom, left, and right offset properties and does not affect other elements.
  • Absolute position can overlap.
  • Elements like headings, paragraphs, divs, and footers are block level by default.
  • Z-index is specified by positive or negative integers.
  • Clear values specify where elements will not touch and include left, right, both, or none.
